GitHub 的 Packages（软件包） 板块。 它之所以会显示这几个名字（如 northflank、yxj20200602、cnmd999），是因为你在该项目中配置并发布了 GitHub Packages（通常是 Docker 镜像或者 npm 包），而这些名字对应的是你发布的不同软件包的名称或标签。 如果你想清理掉这些显示，或者不希望它们出现在右侧边栏，可以通过以下两种方式进行处理： 方法一：彻底删除不需要的软件包（推荐） 如果你这些包只是测试用的，不再需要了，可以直接进入包的管理页面将其彻底删除： 进入软件包列表：点击红框下方的 + 2 个软件包 链接，或者直接在仓库顶部的导航栏（如果是个人主页或组织主页）找到 Packages 标签页。 选择要删除的包：点击进入你想要删除的包（例如 cnmd999）。 进入设置：在软件包的右侧或导航栏中，点击 Package settings（软件包设置）。 彻底删除：滚动到页面底部，在危险区（Danger Zone）找到 Delete this package，输入包名确认删除。 注意：如果这个包被发布为了“公共（Public）”，GitHub 为了防止依赖断裂，有时会限制直接删除。如果无法删除，可以尝试将其设置为“私有（Private）”后再看是否能删，或者使用方法二直接在仓库中隐藏。 方法二：在仓库侧边栏中隐藏「软件包」板块 如果你想保留这些包，但只是不想让它们显示在当前仓库的右侧边栏，可以关闭该板块的显示： 回到当前仓库的主页。 在右侧边栏的顶部（通常在 About（关于） 板块的右上角），有一个齿轮图标（Settings）。 点击齿轮图标，会弹出一个 Layout（布局） 的自定义菜单。 在弹出的选项中，找到 Packages，取消勾选 它。 点击 Save changes（保存修改）。 这样，右侧边栏的“软件包”红框区域就会直接消失，页面会变得更加干净。 帮我写个个人博客，html形式的代码，内容体现热爱网络应用，服务器，这类的，喜欢交朋友 为你定制的个人博客网页代码已经生成完毕。 为了契合你对网络应用、服务器折腾以及喜欢交朋友的特质，我为你设计了一套具有 Geek/Linux 终端黑客风格 兼具 高质感现代设计 的纯 HTML/CSS 代码页。 📄
